# نظام الموسيقى — Madox Bot

## نظرة عامة

نظام الموسيقى يتيح للمستخدمين البحث عن أي أغنية وإرسالها مباشرة في المحادثة.
يعتمد على محرك `musicEngine` المصمم للموثوقية والأداء العالي.

---

## لماذا كان النظام القديم يفشل؟

| المشكلة | التأثير |
|---------|---------|
| `execFileSync` و`execSync` | يُجمّد كامل البوت أثناء التحميل |
| تحميل yt-dlp عبر `curl` (متزامن) | يُجمّد البوت لدقيقتين عند أول استخدام |
| مزود بحث واحد فقط (yt-search) | أي خطأ = فشل كامل |
| لا queue لإدارة الطلبات | طلبات متعددة تُشبع الذاكرة |
| لا تنظيف للملفات المؤقتة عند الإعادة | تراكم الملفات في التخزين |
| لا cooldown للمستخدم | إمكانية إرسال عشرات الطلبات بلحظة |

---

## كيف يعمل النظام الجديد؟

### سلسلة المزودين

```
المستخدم: -music [اسم الأغنية]
    ↓
[1] YouTube Search (yt-search)
    ↓ إذا نجح → yt-dlp async spawn → ملف صوتي كامل
    ↓ إذا فشل
[2] iTunes Search API (مجاني، بدون مفتاح)
    ↓ إذا نجح → تحميل مباشر → معاينة 30 ثانية
    ↓ إذا فشل
[رسالة خطأ واضحة]
```

### المراحل

```
1. البحث    → yt-search + iTunes fallback
2. الإخطار  → إرسال "وجدتها: [العنوان]، جاري التحميل..."
3. التحميل  → spawn async (لا يُجمّد البوت)
4. التحقق   → فحص الملف (الحجم، الوجود)
5. الإرسال  → مع timeout 90 ثانية
6. التنظيف  → حذف الملف بعد 30 ثانية
```

---

## الأوامر المتاحة

| الأمر | الوصف |
|-------|-------|
| `-music [اسم]` | البحث عن أغنية وإرسالها |
| `-song [اسم]` | نفس الأمر (اسم بديل) |
| `-music diag` | عرض حالة المحرك (للمشرف) |

---

## ضوابط الاستخدام

| الضابط | القيمة |
|--------|--------|
| أطول مدة مقبولة | 12 دقيقة |
| أكبر حجم ملف | 48 ميجابايت |
| cooldown بين الطلبات | 35 ثانية / مستخدم |
| طلبات متزامنة | 2 في نفس الوقت |
| أقصى حجم قائمة انتظار | 5 طلبات |
| انتهاء مهلة التحميل | 110 ثانية |

---

## استكشاف الأخطاء

### الأغنية لا تُرسَل

1. تحقق من `-music diag` لعرض الحالة
2. تأكد أن yt-dlp مثبت: `yt-dlp --version`
3. جرّب أغنية مختلفة

### "انتهت مهلة التحميل"

- الشبكة بطيئة أو YouTube يُحاول الحجب
- انتظر دقيقة ثم حاول مجدداً
- جرّب أغنية أقصر مدة

### "قائمة الانتظار ممتلئة"

- هناك 5 طلبات نشطة حالياً
- انتظر دقيقة

### "لم يُعثر على نتائج"

- جرّب كتابة الاسم بالإنجليزية
- أضف اسم الفنان: `-music شاكيرا - وكا وكا`

---

## تنظيف الملفات المؤقتة

الملفات المؤقتة تُخزَّن في:
```
/tmp/madox_music/music_*.m4a
```

تُحذف تلقائياً:
- بعد 30 ثانية من الإرسال
- عند تشغيل البوت (الملفات أكبر من ساعة)

للتنظيف اليدوي:
```bash
rm -f /tmp/madox_music/music_*.m4a
```

---

## إعداد بيئة الإنتاج (Railway)

يتم تثبيت yt-dlp تلقائياً في مرحلة البناء عبر:
```toml
[phases.build]
cmds = ["pip3 install -q --no-cache-dir yt-dlp"]
```

لا داعي لأي إعداد يدوي.
